## SHIP / DON'T SHIP verdict

**Already shipped** (commit `baa2f0f` on `main`). This audit is post-ship.

Treat the findings below as **follow-up work items** rather than merge blockers. All are in-scope for a hotfix commit.

### P0 (stop-the-line — fix before any external submission / launch review)

1. **TypeScript errors in compliance UI + API** — 16+ Supabase type mismatches. Fix: `npx supabase gen types typescript --project-id nnhkcufyqjojdbvdrpky > src/lib/supabase/types.ts` (or the project's type-gen path), then verify with `npx tsc --noEmit`. **Mechanical; ~5 min.**

2. **Kelsey not in Arnold coaching pipeline.** `src/lib/body-tracker/arnold-recommender.ts` renders + persists recommendations without calling Kelsey. **This is the single biggest regulatory risk surface** — Claude-voiced free-form text is exactly the channel most likely to leak unapproved disease claims. Fix: wrap the Claude API call in a Stage-1 detector + Stage-2 Kelsey verdict gate.

3. **Peptide database missing BPC-157 / Epithalon / TB-500.** 3 of 9 spec-required peptides absent from `src/config/peptide-database/peptides-54b.ts`. The `peptide_classifications` table has the regulatory records but the consumer-facing catalog does not carry the products. Fix: seed the 3 missing peptides or confirm they are intentionally excluded.

4. **NPN / DIN-HM validator absent.** Columns exist in `regulatory_sku_compliance`; no validator function gates writes. A malformed NPN can land in the DB. Fix: add a Zod / regex validator + API route middleware.

5. **Kelsey → Sherlock / Hannah / Arnold escalation contracts undefined.** Kelsey verdicts flag escalations in prose but don't emit structured events. Fix: add a `kelsey_escalations` channel + per-agent subscriber contracts.

### P1 (pre-launch cleanup)

1. **Only 1 of 4 new regulatory guardrails in `src/lib/agents/jeffery/guardrails.ts`.** Bioavailability range is present. DSHEA / jurisdiction / disease-claim guardrails need to be composed into the Jeffery guardrail list (or explicitly noted as out-of-scope for that file).

2. **Alert-feed ingestion has no schema validation before persistence.** `supabase/functions/compliance-alerts-daily-sync/index.ts` trusts upstream FDA/HC/FTC payloads. Add Zod validation to each source adapter.

3. **CA (Canadian) users have no NHPID replacement disclaimer + no French parity.** DSHEA correctly hidden; nothing replaces it. French-language surfaces required for Quebec compliance when the practitioner channel expands.

4. **Jurisdiction gate not in HTTP middleware.** Only enforced at DB layer + one UI component. Add to `src/middleware.ts` for defense-in-depth.

5. **Golden-claim regression suite is 24 cases, not 120.** Per commit message. Expand the set before any external regulatory review cites the test coverage.
